Scope

Technical

Platform Overview

The regime platform combines statistical state detection, rules-based signal generation, qualitative frontier overlays, and trading workflows in one FastAPI app.

Regime engine

Decision layer

Decision rule: the dashboard never trusts a single subsystem. Quantitative outputs, qualitative overlays, tax data, and broker status are kept separate so failures degrade instead of cascading.

Technical

HMM Regime Engine

The regime engine uses a 3-state Hidden Markov Model with adaptive fit windows and diagonal covariance to avoid the look-ahead bias removed in Sprint 20.

ComponentCurrent rule
States3 latent states mapped to Bull, Neutral, Bear
Features6-feature set after collinear feature removal
CovarianceDiagonal
Fit strategyWalk-forward with adaptive training-window retries on shorter histories
Weekly overlayResampled weekly bars with fallback when insufficient rows exist
Important: a non-convergence log from hmmlearn is not automatically a hard failure. The engine treats completed fits and insufficient-history failures differently.
Confidence basis

Unified confidence is derived from state probability, transition stability, and calibration adjustments rather than raw HMM probability alone.

Technical

Signal Rules Engine

The signal layer converts regime forecasts into actions, targets, warnings, and position sizes using explicit thresholds and downstream guardrails.

Primary action rules
ActionTypical conditionsOutput
Strong BuyHigh Bull probability, low transition risk, strong duration supportAggressive entry, full targets, sizing suggestion
BuyBull-tilting conditions without strongest gateEntry zone, ATR stop, moderate sizing
HoldMixed or stable neutral conditionsNo new entry target, monitoring stop
Sell / Strong SellBear state, adverse transition, or strong downside probabilitiesExit framing, risk-off guidance
Price target model

ATR projected move = ATR × √time × confidence multiplier × mean-reversion dampening

Position sizing model

Suggested size = min(regime confidence, half-Kelly cap, max-risk cap)

Conflict rule: if action and risk/reward disagree, the engine surfaces an explicit warning instead of silently favoring one side.

The qualitative pipeline fetches recent catalysts, sentiment context, and thesis/theme context, then asks a configured frontier provider for institutional-style commentary.

Technical

Deployment & Operations

Investor runs as a FastAPI app with local static assets, scheduled jobs, broker polling, and environment-driven provider/broker configuration.

  1. Start the FastAPI app from the Investor project root so relative data paths resolve correctly.
  2. Run scheduled jobs from the same environment as the app to share caches, persistence, and broker settings.
  3. Prefer route-level graceful degradation for external failures instead of process-level aborts.
Operational rule: if a full suite run fails from /Volumes/T9 but passes from the Investor root, treat it as an invocation-path issue before debugging code.

User Guide

Quick Start

Use this sequence when opening the dashboard cold.

  1. Select the benchmark, period, portfolio scope, and account filter.
  2. Pick holdings from the searchable list or add custom symbols.
  3. Enable Frontier only if you want qualitative analysis and the extra latency is acceptable.
  4. Run analysis and watch partial ticker cards stream in as the job progresses.
  5. Move to Trading only after reviewing action items and quantitative warnings in Analysis.

User Guide

User Guide

Running Regime Analysis

The job runner is asynchronous. The page shell loads immediately, then partial ticker results arrive as the background thread completes daily fits and optional frontier analysis.

Status bar meaning: percentage completion is approximate when frontier analysis is enabled because cached qualitative hits and live LLM requests have different runtimes.
  1. Scope the universe with portfolio and account filters.
  2. Use “Show all holdings” only when you explicitly want historical breadth beyond current snapshots.
  3. Add custom symbols as comma, semicolon, newline, or whitespace-separated tickers. Share-class inputs like BRK B remain intact.

User Guide

Reading Signals

The detail card is intentionally layered: math first, frontier second, charts below.

Math panel

State stats, expected duration, volatility, price targets, sizing, earnings context.

Frontier panel

Thesis validation, catalysts, divergence, verdict, urgency, risk trigger.

Warnings

Risk/reward conflict, stop proximity, earnings risk, regime disagreement, tax drag.

User Guide

Paper Trading Workflow

Paper portfolios are review-first. The scheduler generates plans; execution remains an operator choice unless explicitly automated later.

  1. Create or select a paper portfolio.
  2. Generate plans to pull discovery entries and exit conditions into the review queue.
  3. Approve, reject, or modify plans after precheck badges and guardrails render.
  4. Execute approved trades and review the audit trail plus performance attribution.

Tax & Lot Interpretation

Lot drill-downs reflect currently held lots from the scoped account snapshot path, not broad historical lots, unless the operator explicitly requests a broader view.

Active-lot rule: dust and exited positions should be filtered out by the current-holdings ORM path. If they appear, treat it as a data-path bug, not user error.
FieldInterpretation
Tax statusSummarized as counts such as 2 ST · 3 LT
Days to LTCGDays until a short-term lot crosses into long-term treatment
Action impactShould compress to a dash when no material tax adjustment exists
