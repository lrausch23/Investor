Core Principles

Local-First

SQLite databases travel with the project directory. No external DB server required.

Audit-First

Every mutation logged to order_audit_trail. Full decision provenance.

Explainability

Every signal, verdict, and action is traceable to specific thresholds and rules.

Tax-Aware

LTCG protection, wash-sale detection, and tax-loss harvesting built into the decision pipeline.

Cache-First

TTL-gated caches for market data, earnings, crowd scores, and qualitative analysis.

Safe-by-Default

Guardrails block trades that exceed position, order, loss, or exposure limits.

Decision rule: the dashboard never trusts a single subsystem. Quantitative outputs, qualitative overlays, tax data, and broker status are kept separate so failures degrade instead of cascading.

HMM Regime Engine

The regime engine uses a 3-state Hidden Markov Model with adaptive fit windows and diagonal covariance to avoid the look-ahead bias removed in Sprint 20.

Feature Engineering

ReturnDaily log return: ln(price_t / price_{t-1})1 day
VolatilityAnnualized: rolling_std(return, 20) × √25220 days
TrendSMA ratio: price / SMA(20) − 120 days
Volume Z-Score(volume − rolling_mean) / rolling_std20 days
VIX ChangeDaily change in CBOE VIX index1 day
10Y Yield ChangeDaily change in US 10-Year Treasury yield1 day
Macro weighting: VIX and yield features receive a 1.5× weighting multiplier when macro inputs are enabled.

States3 latent states mapped to Bull, Neutral, Bear

CovarianceDiagonal

Signal Rules Engine

The signal layer converts regime forecasts into actions, targets, warnings, and position sizes using explicit thresholds and downstream guardrails.

Stage 1: Forward Regime Curve

forward_regime_curve() — Projects state probabilities 21 days forward via Markov matrix exponentiation.

Formula: P(state, t+k) = P(state, t) × T^k where T is the transition matrix and k is the forecast horizon (1–21 days).
Stage 2: Forward Signal

signal_from_forward_curve() — Maps projected probabilities to action signals.

ConditionSignalRequirements
Bull, low risk, stable, high probStrong Buytrans_risk < 0.05, duration > 15d, p_bull_5d ≥ 0.70
Bull, moderate riskBuytrans_risk < 0.15
Neutral, tilting bullishBuyp_bull_5d > 0.40
Bear, low risk, stable, high probStrong Selltrans_risk < 0.05, duration > 10d, p_bear_5d ≥ 0.70
Bear, moderate riskSelltrans_risk < 0.15
Bull/Neutral with emerging bearSellp_bear_5d > 0.40
All otherHold—
Earnings adjustment: Signal strength reduced by 0.15 when earnings fall within the holding period.

Stage 4: Intra-Regime Signal

Decision table by regime × condition → tactical signal. Examples:

RegimeConditionSignal
BullRSI oversold + price near BB lowerBuy the dip
BullRSI overbought + extended above BB upperTake profits
BearRally into resistanceSell the rally

Stage 5: Composite Signal

Merges regime (forward) signal with technical (intra-regime) signal. Conflict resolution:

Forward SignalTechnical SignalResolution
Buy / Strong BuyBuy the dipStrength +15–20% (confirmation)
Buy / Strong BuyTake profitsOverride → Hold (divergence)
Buy / Strong BuyStay defensiveStrength −20% (conflict)
Sell / Strong SellSell the rallyStrength +15–20% (confirmation)
Sell / Strong SellCover shortOverride → Hold (divergence)
HoldAnyTechnical used as tiebreaker
Stage 6: Price Targets & Stops
ATR Projection: target = price ± (ATR × √holding_period × confidence_multiplier × reversion_factor)
Target TypeDerivation
EntryBollinger Band proximity + regime direction
ExitATR projected move from entry
Stop2 × ATR below entry (long) or above entry (short)
Stage 7: Position Sizing
Half-Kelly: kelly_fraction = (win_prob × avg_win − loss_prob × avg_loss) / avg_win
position_size = 0.5 × kelly_fraction (capped by portfolio constraints)

Portfolio adjustments: regime exposure limits, sector concentration caps, correlation penalties.

Stage 8: Confidence Score
Formula: confidence = (regime_probability × 0.70 + signal_strength × 0.30) × 100
Score RangeTierAction Implication
80–100HighFull position sizing
60–79MediumReduced sizing
40–59LowMonitor only
<40Very LowNo action
Stage 9: Confidence Trajectory

10-day linear regression on confidence scores. Slope thresholds: ±0.005.

SlopeLabel
> +0.005Improving
−0.005 to +0.005Stable
< −0.005Deteriorating
Stage 10: Sentiment Momentum

VADER compound score on 30-day news window. Divergence detection:

RegimeSentimentFlag
BullDeterioratingBearish divergence — caution
BearImprovingBullish divergence — potential reversal
Price target model

ATR projected move = ATR × √time × confidence multiplier × mean-reversion dampening

Position sizing model

Suggested size = min(regime confidence, half-Kelly cap, max-risk cap)

Conflict rule: if action and risk/reward disagree, the engine surfaces an explicit warning instead of silently favoring one side.

Deployment & Operations

Investor runs as a FastAPI app with local static assets, scheduled jobs, broker polling, and environment-driven provider/broker configuration.

  1. Start the FastAPI app from the Investor project root so relative data paths resolve correctly.
  2. Run scheduled jobs from the same environment as the app to share caches, persistence, and broker settings.
  3. Prefer route-level graceful degradation for external failures instead of process-level aborts.
Operational rule: if a full suite run fails from /Volumes/T9 but passes from the Investor root, treat it as an invocation-path issue before debugging code.

User Guide

Running Regime Analysis

The job runner is asynchronous. The page shell loads immediately, then partial ticker results arrive as the background thread completes daily fits and optional frontier analysis.

Status bar meaning: percentage completion is approximate when frontier analysis is enabled because cached qualitative hits and live LLM requests have different runtimes.
  1. Scope the universe with portfolio and account filters.
  2. Use “Show all holdings” only when you explicitly want historical breadth beyond current snapshots.
  3. Add custom symbols as comma, semicolon, newline, or whitespace-separated tickers. Share-class inputs like BRK B remain intact.

Three Regime States

Bull

Uptrend detected. Typical signals: Buy / Strong Buy. Position sizing at full allocation.

Neutral

No clear trend. Typical signals: Hold. Monitor for bullish or bearish tilt.

Bear

Downtrend detected. Typical signals: Sell / Strong Sell. Risk-off positioning.

"Regime Too New" Suppression

Configurable minimum days: Streamlit slider default is 3, range 1–10. Laszlo's preferred setting: 5 days.

When regime_days < min_regime_days, the AI Verdict is suppressed to "Hold — Regime too new (Xd < Yd minimum)" and the Action column displays "—" (blank). This prevents whipsaw trades on false regime transitions.

User Guide

Paper Trading Workflow

Paper portfolios are review-first. The scheduler generates plans; execution remains an operator choice unless explicitly automated later.

  1. Create or select a paper portfolio.
  2. Generate plans to pull discovery entries and exit conditions into the review queue.
  3. Approve, reject, or modify plans after precheck badges and guardrails render.
  4. Execute approved trades and review the audit trail plus performance attribution.

Tax & Lot Interpretation

Lot drill-downs reflect currently held lots from the scoped account snapshot path, not broad historical lots, unless the operator explicitly requests a broader view.

Active-lot rule: dust and exited positions should be filtered out by the current-holdings ORM path. If they appear, treat it as a data-path bug, not user error.
